# 树莓派客户端使用说明

## 概述

树莓派客户端现在使用 PyQt5 桌面应用程序界面，替代了原有的网页界面。新界面包含：

1. **视频通话功能**
   - 本地摄像头画面显示（小窗口）
   - 远程医生视频画面显示（大窗口）
   - 音频通话开关控制

2. **实时聊天功能**
   - 发送消息给医生
   - 接收医生的消息
   - 消息气泡式显示

## 安装依赖

在树莓派上运行以下命令安装所需依赖：

```bash
# 安装系统依赖（PyQt5 需要）
sudo apt-get update
sudo apt-get install -y python3-pyqt5 python3-pyqt5.qtwebkit

# 安装 Python 依赖
cd /path/to/代码1
pip3 install -r requirements.txt
```

## 启动客户端

### 方法1：使用启动脚本

```bash
cd /path/to/代码1/client
python3 run_client.py
```

### 方法2：直接运行

```bash
cd /path/to/代码1/client
python3 gui_client.py
```

## 配置说明

编辑 `config.py` 文件修改服务器地址：

```python
SERVER_HOST = '192.168.185.61'  # 修改为服务器实际IP地址
SERVER_PORT = 5000
```

## 界面说明

### 左侧视频区域
- **上方大窗口**：显示医生的实时视频画面
- **下方小窗口**：显示本地摄像头画面
- **音频按钮**：点击开启/关闭音频通话

### 右侧聊天区域
- **聊天记录**：显示医患之间的对话历史
- **输入框**：输入要发送的消息
- **发送按钮**：发送消息

## 快捷键

- `Enter`：在输入框中按 Enter 快速发送消息
- `Alt+F4` 或点击关闭按钮：退出应用程序

## 注意事项

1. 确保树莓派已连接摄像头
2. 确保树莓派和服务器在同一网络下
3. 首次启动可能需要几秒钟初始化摄像头
4. 如果摄像头初始化失败，会显示默认黑屏

## 故障排除

### 摄像头无法启动
```bash
# 检查摄像头是否被识别
ls /dev/video*

# 测试摄像头
raspistill -v -o test.jpg
```

### 音频问题
```bash
# 检查音频设备
arecord -l
aplay -l
```

### 连接服务器失败
- 检查服务器 IP 地址配置是否正确
- 确保服务器已启动并运行
- 检查网络连接
